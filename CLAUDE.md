# CLAUDE.md

Этот файл — постоянный контекст для Claude Code при работе над дипломной работой (бакалавриат, ВКР). Перед любыми правками `.tex`, структуры или библиографии — свериться с этим файлом.

## О проекте

- **Тип работы:** Выпускная квалификационная работа бакалавра (ВКР).
- **Инструмент сборки:** LaTeX.
- **Язык работы:** русский.
- **Подробные требования к оформлению:** см. [`requirements.md`](./requirements.md) — это «единственный источник истины» по форматированию. Любое расхождение между `requirements.md` и моим решением — побеждает `requirements.md`.

## Соглашения проекта

### Именование

- Файлы и директории: **kebab-case** (например, `chapter-1-introduction.tex`, `images/architecture-diagram.pdf`, `bib/sources.bib`).
- Исключения: общепринятые имена (`CLAUDE.md`, `README.md`, `Makefile`, `main.tex` — если они появятся).

### LaTeX

- Шрифт: **Times** или **Computer Modern** (с засечками). Не использовать sans-serif для основного текста.
- Базовый размер шрифта: **14pt** (например, `\documentclass[14pt]{extarticle}` или `extreport`).
- Межстрочный интервал: **1.5** (например, через `\usepackage{setspace}\onehalfspacing`).
- Поля: верх/низ — 2 см, лево — 3 см, право — 1 см (`\usepackage[top=2cm,bottom=2cm,left=3cm,right=1cm]{geometry}`).
- Абзацный отступ: **1.25 см** (`\setlength{\parindent}{1.25cm}`), одинаковый по всему документу.
- Выравнивание: **по ширине** (LaTeX по умолчанию).
- Текст: **только чёрный**. Не использовать цвета в основном тексте.
- Номера страниц: **внизу по центру**, **сквозная** нумерация; на титульном листе номер не печатается, но он считается первой страницей.

### Заголовки

- Названия глав, разделов, параграфов — строчными с первой прописной, **полужирные**, выравнивание по левому краю с абзацным отступом.
- В конце **любого заголовка точка НЕ ставится**. Точка ставится только в конце **номера** раздела/секции/главы.
- «Содержание», «Введение», «Заключение», «Приложение» — с прописной буквы, по левому краю, с абзацным отступом.
- Каждая новая глава, а также введение/заключение/список литературы/приложения — **с новой страницы**.
- Параграфы и пункты — **по тексту, без принудительных переносов**.

### Таблицы, рисунки, формулы

- Подпись таблицы — **сверху, справа**, шрифт **12pt**, формат: `Таблица N. Название`.
- Подпись рисунка — **снизу, по центру**, формат: `Рис. N. Название`.
- Нумерация таблиц и рисунков — **сквозная** до приложений; в приложениях — отдельная с префиксом (`Таблица В.1`).
- На каждый рисунок и таблицу должна быть **ссылка в тексте**.
- Формулы: нумеруются только выключенные формулы, на которые есть ссылка. Номер — справа, прижат к правому краю. Ссылка из текста — в круглых скобках: `(1)`. Формулы внутри текста не нумеруются.

### Списки

Допустимы два стиля (не смешивать в пределах одного списка):

1. Пункты с прописной, после цифры точка, в конце пункта точка.
2. Пункты со строчной, после буквы скобка (`а)`, `б)`...), в конце пункта точка с запятой; последний пункт — точкой.

### Ссылки на источники

- Пересказ: `[15]`.
- Прямая цитата: `[14, с. 236]`.
- Группа источников: `[1-6, 18, 21, 34]`.

### Библиография

- Стандарт: **ГОСТ 7.0.5-2008**.
- Раздел: «Список литературы», нумерованный список (арабские цифры с точкой).
- Порядок: по алфавиту **или** по порядку цитирования (выбрать один и не смешивать).
- Иностранные источники — **в конце**.
- Для BibTeX/biblatex использовать стиль, совместимый с ГОСТ (например, `biblatex-gost`).
- На каждый источник в списке **обязательна ссылка в тексте**.

### Приложения

- Обозначаются заглавными русскими буквами: А, Б, В, Г, Д, Е, Ж, И, К, Л, М, Н, П, Р, С, Т, У, Ф, Х, Ц, Ш, Щ, Э, Ю, Я.
- **Пропускаются:** Ё, З, Й, О, Ч, Ь, Ы, Ъ.
- Ссылка в тексте: `(см. приложение Б)`.

## Чего избегать

- Не менять формат (поля, шрифт, отступы) под отдельные части документа — всё должно быть однородным.
- Не использовать цветной текст в основной части.
- Не ставить точку в конце заголовков.
- Не использовать sans-serif шрифты для основного текста.
- Не нумеровать формулы, если на них нет ссылки.
- Не смешивать стили списков и стили библиографии.
- Не добавлять в проект «полезные» абстракции, шаблоны, скрипты, которых пользователь не просил.

## Стиль работы

- Перед существенными изменениями структуры или конфигурации сборки — спросить пользователя.
- Все изменения в `.tex` файлах — атомарные, в рамках одной правки.
- Сомневаешься в форматировании — сверься с `requirements.md`, потом задавай вопрос.
