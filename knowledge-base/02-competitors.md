# 02. Конкуренты

Анализ конкурентного поля основан на внутреннем инвестиционном обзоре проекта (документ «ВКурсе. Инвестиционный обзор», версия 2.0 от 8 мая 2026 года). Конкурентное окружение делится на два класса: прямые конкуренты — системы управления учебным расписанием на стороне вуза, и непрямые — мобильные приложения-агрегаторы, парсящие публичные источники.

## Прямые конкуренты

### Галактика РУЗ

Модуль управления учебным расписанием в составе ERP-экосистемы «Галактика». Ориентирована преимущественно на крупные государственные вузы.

| Параметр | Значение |
|----------|----------|
| Модель поставки | On-premise (локальная установка) |
| Мобильные приложения | Отсутствуют |
| Интеграция | Ограничена экосистемой «Галактика» |
| Стоимость внедрения | Высокая; разовый платёж + обслуживание |
| Целевая аудитория | Крупные государственные вузы |

**Ограничения:** монолитная архитектура, высокая стоимость, отсутствие современной мобильной составляющей, длительный цикл внедрения.

### ММИС Лаб

Российский вендор ПО для автоматизации учебного процесса (управление контингентом, нагрузкой, расписанием).

| Параметр | Значение |
|----------|----------|
| Модель поставки | On-premise |
| Мобильные приложения | Отсутствуют; веб-интерфейс адаптирован частично |
| Интеграция | Ограниченный и неконсистентный API |
| Стоимость внедрения | Средняя–высокая |
| Целевая аудитория | Средние и крупные вузы |

**Ограничения:** нет нативных мобильных приложений, медленный темп обновлений, высокий порог входа.

### 1С:Университет

Отраслевое решение на базе «1С:Предприятие». Модуль расписания — один из компонентов общей системы.

| Параметр | Значение |
|----------|----------|
| Модель поставки | On-premise на базе «1С:Предприятие» |
| Мобильные приложения | Отсутствуют |
| Интеграция | Преимущественно внутри экосистемы 1С |
| Стоимость внедрения | Очень высокая; ориентир — от 110 тыс. руб./мес |
| Целевая аудитория | Госвузы с инфраструктурой 1С |

**Ограничения:** жёсткая зависимость от «1С:Предприятие», сложный для академического персонала интерфейс, отсутствие нативных мобильных клиентов.

## Сводное сравнение прямых конкурентов

| Критерий | ВКурсе | Галактика РУЗ | ММИС Лаб | 1С:Университет |
|----------|--------|---------------|----------|-----------------|
| Модель поставки | SaaS (облако) | On-premise | On-premise | On-premise |
| Нативные мобильные приложения | Да | Нет | Нет | Нет |
| Современный UI/UX | Да | Нет | Нет | Нет |
| Быстрое подключение | Да | Нет | Нет | Нет |
| Открытый API | Да | Ограничен | Ограничен | Отсутствует |
| Независимость от сторонней ERP | Да | Нет | Да | Нет |
| Низкий порог входа по стоимости | Да | Нет | Нет | Нет |

## Непрямые конкуренты — приложения-парсеры

Самый показательный представитель — приложение «Кампус» и аналогичные. Не управляют расписанием, а агрегируют его, парся публичные сайты учебных заведений.

**Бизнес-модель:** реклама в интерфейсе + опциональная подписка для отключения рекламы.

**Системные ограничения:**

- **Нестабильность и неполнота покрытия.** Любое изменение вёрстки или URL-структуры сайта-источника приводит к отказу парсера.
- **Отсутствие реального времени.** Оперативные изменения (замены, переносы, новые аудитории) отображаются с задержкой или не отображаются вовсе.
- **Качество интерфейса.** Реклама, баннеры, интерстишал — снижают UX и делают продукт непригодным для официального позиционирования со стороны вуза.
- **Отсутствие институциональной легитимности.** Вуз не является клиентом и не несёт ответственности за корректность данных.
- **Односторонняя функциональность.** Только просмотр; нет официальных уведомлений, нет инструментов для администрации.

## Сравнение с приложениями-парсерами

| Критерий | ВКурсе | Приложения-парсеры |
|----------|--------|---------------------|
| Источник данных | Прямая интеграция с вузом | Парсинг публичных сайтов |
| Актуальность | Реальное время | Задержки, возможны ошибки |
| Охват вузов | Подключённые партнёры | Нестабильно |
| Управление расписанием для администратора | Есть | Нет |
| Рекламная монетизация | Нет | Да; основной источник дохода |
| Официальное партнёрство с вузом | Да | Нет |
| Надёжность при изменении источника | Работа в собственной экосистеме | Низкая |
| Институциональная ответственность за данные | Да | Нет |

## Историческая ремарка

В начале проекта (осень 2024 — зима 2026) сама команда «ВКурсе» работала по парсинговой модели: подключала API/парсинг расписаний шести омских вузов (см. [pilots-and-partners.md](pilots-and-partners.md)). Этот опыт позволил на практике оценить ограничения подхода и принять стратегическое решение в пользу собственной SaaS-платформы.

## Вывод

Российский рынок систем управления учебным расписанием характеризуется преобладанием on-premise решений, высокой зависимостью от крупных ERP-экосистем и отсутствием полноценной мобильной составляющей у прямых конкурентов. Парсинговые приложения подтверждают спрос на мобильный доступ, но не способны закрыть потребность в официальной, управляемой и надёжной платформе. «ВКурсе» закрывает обе ниши: SaaS-модель с предсказуемой стоимостью, нативный мобильный опыт, независимость от сторонних ERP, прямая интеграция с вузом.
