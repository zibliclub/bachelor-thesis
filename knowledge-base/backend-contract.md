# Backend: то, что мы знаем со стороны клиентов

Бэкенд разрабатывается соавтором (Максим, CTO) в рамках отдельной ВКР. В рамках текущей работы бэк рассматривается **только в объёме интеграционного контракта**.

## Что известно сейчас

- **Язык:** Java.
- **Фреймворк:** Spring (предположительно Spring Boot, подтвердить).
- **Аналитика:** Grafana. Метрики: количество запросов по конкретным вузам, распределение по платформам (iOS/Android/Web), нагрузка на сервер. Подсчёт уникальных и новых пользователей через UUID, генерируемый клиентом при первом запуске и отсылаемый с каждым запросом.
- **Парсинговая модель:** клиент запрашивает расписание → бэк обращается к сайту вуза, преобразует в единый формат → отдаёт клиенту.

## Что неизвестно и требует уточнения

Полный список вопросов — в [pending-questions-backend.md](pending-questions-backend.md). Ключевые открытые темы:

- Точный стек: версии Java и Spring, СУБД, кеш, очереди.
- Формат API: REST / GraphQL / gRPC; наличие OpenAPI/Swagger.
- Аутентификация в SaaS-сервисе (JWT / OAuth2 / refresh).
- Доставка обновлений расписания в реальном времени (pull / WebSocket / SSE / push).
- Push-уведомления на клиенты (APNs / FCM).
- Хостинг и деплой.

После опроса Максима факты переезжают сюда из `pending-questions-backend.md`.
