# Открытые вопросы к Максиму по бэку

Этот файл — список вопросов, ответы на которые нужны, чтобы корректно описать клиент-серверный контракт в разделах 2.1.2 и 3 ВКР. После получения ответов факты переезжают в [backend-contract.md](backend-contract.md), а этот файл сокращается.

## 1. Точный технический стек

- [ ] Версия Java (17? 21?).
- [ ] Spring Boot или просто Spring? Версия.
- [ ] Сборщик: Maven или Gradle.
- [ ] СУБД: PostgreSQL / MySQL / иное. Версия.
- [ ] Кеш: Redis / встроенный / нет.
- [ ] Очереди / брокер сообщений: RabbitMQ / Kafka / нет.

## 2. API

- [ ] Тип API: REST, GraphQL, gRPC, смешанный.
- [ ] Формат сериализации (JSON / Protobuf).
- [ ] Спецификация: есть ли OpenAPI/Swagger? Если да — URL или путь к файлу.
- [ ] Версионирование API: путь (`/v1/`), header, иное.
- [ ] Пагинация для списков (cursor / page+size).

## 3. Аутентификация и авторизация

- [ ] Используется ли JWT? OAuth2? Сессионные cookie?
- [ ] Refresh-токены: схема, время жизни.
- [ ] Ролевая модель для веб-админки (admin / employee) — где реализована, как клиенту приходит роль.
- [ ] Multi-tenancy: разделение данных по учебным заведениям.

## 4. Парсинг расписаний

- [ ] Триггер парсинга: cron / ручной / по запросу клиента.
- [ ] Хранение промежуточных данных (raw HTML/PDF, разобранные структуры).
- [ ] Стратегия обработки сбоев: ретраи, уведомления, fallback на кеш.
- [ ] Как клиент узнаёт о том, что данные устарели или недоступны.

## 5. Реальное время и доставка обновлений

- [ ] Pull (HTTP polling) / WebSocket / Server-Sent Events / push.
- [ ] Если push — APNs/FCM, конфигурация, темы, маршрутизация.
- [ ] ETag / If-Modified-Since / собственный механизм инвалидации кеша.

## 6. Бэкенд SaaS-сервиса (не парсинг)

- [ ] Структура данных: учебные заведения, пользователи, группы, преподаватели, аудитории, факультеты, кафедры, корпуса, расписание.
- [ ] ER-диаграмма (хотя бы черновая).
- [ ] Валидация коллизий: на каком уровне (БД constraints / прикладной слой), какие именно правила.
- [ ] Аудит-лог изменений расписания.

## 7. Инфраструктура

- [ ] Хостинг: где сервер (свой/VPS/облако), ОС.
- [ ] Контейнеризация (Docker, Docker Compose, Kubernetes).
- [ ] Reverse proxy (nginx / Caddy / traefik).
- [ ] TLS / Let's Encrypt.

## 8. Аналитика и мониторинг

- [ ] Grafana: поверх чего работает (Prometheus, Loki, иное).
- [ ] Какие метрики собираются (RED, USE, бизнес-метрики).
- [ ] Какие логи и где хранятся.
- [ ] Алерты: настроены ли, на что.

## 9. Тестирование

- [ ] Есть ли юнит-тесты, интеграционные, контрактные.
- [ ] Покрытие (если есть число).

## 10. CI/CD

- [ ] Что используется (GitHub Actions, GitLab CI, иное).
- [ ] Как происходит деплой на сервер.
- [ ] Стратегия миграций БД (Flyway / Liquibase).
