# Инфраструктура и CI/CD клиентов

## iOS

- Сборка релизов: Xcode Archive → загрузка в App Store Connect вручную.
- Ранее использовался Xcode Cloud, но он был ограничен на территории России в 2025 году. Команда переключилась на ручной флоу.

## Android

- Публикация в Google Play и RuStore.
- (Конкретный pipeline — Gradle Play Publisher, fastlane, GitHub Actions — уточнить при ревью репо.)

## Web

- Push в ветку → подключение к серверу по SSH → запуск Docker-сборки → пользователи получают обновлённую версию.
- Это упрощённая схема. В планах — переход на более зрелый pipeline (CI/CD без ручного SSH).

## Аналитика

- Бэкенд: Grafana. Видна нагрузка по серверу и распределение по вузам/платформам.
- Уникальные пользователи: UUID, генерируемый клиентом при первом запуске и отсылаемый с каждым запросом.

## Тестирование

**Главное ограничение текущей реализации:** автоматических тестов на клиентах **нет**. Это сознательная отговорка из-за приоритизации продуктовой скорости, но в дипломе оформляется как открытое ограничение и пункт в перспективах развития. Аргументация в работе:

- Команда из двух человек, ограниченный таймлайн, акцент на доведении продукта до пилотов.
- Качество подтверждается продакшен-эксплуатацией (~1000 уникальных пользователей в день) и пользовательской обратной связью через Telegram-канал.
- Введение тестов запланировано после стабилизации API SaaS-сервиса (после первого пилота). Приоритеты: контрактные тесты сетевого слоя на iOS/Android и e2e на критических флоу веб-админки.
